Mobile App

WS 2023/24

In one university module we learned how to code a mobile app. The theme for that task was "a chat app with a twist". So we implemented in a group of three a morse code chat app.

For this project we should use flutter to code, thankfully flutter hat a lot of packages we sadly underestimated. In the end of this project we learned how much easier the implementation could have been with more packages. But we are still glad to could make this experience of coding a chat app.
Interesting functions of our app would be the name automatic name suffix that is generated when creating an account. 4 random numbers (that are in that combination for a chosen username not already used) will be generated and attached to ones username.
For people that aren't proficient in morse code there is a little tutorial for a few letters. Other than that there is a cheat sheet in each chat to look up the different morse codes. You can either translate or read out loud (beeping) all text messages as an extra. If you would rather write in clear text, you can do so as well and change the morse keyboard to a regular keyboard. But all communication will be sent as morse code. In addition to that, there aren't only one-on-one chats, but you can create chat groups as well with friends of your contact list.
We used Firebase as our server to safe messages and user data. Because of Firebase, we gladly had less issues with backend communication, because the documentation of firebase was detailed and easy to understand.

Music Visualizer

SS 2019

This university project represents a music visualizer and was made in java based processing.

This program takes a music file as input and analyzes its frequencies. Based on the values of bass, mid and high frequencies, symbols and geometry are formed.
The main visualization is a big triangle that moves to the frequencies. Each edge of the triangle represents either bass, mid or high frequency values. In the background there are different things to see. Depending on the chosen display modus, the user can see pyramids, spheres, rectangle planes or arches in the background. These "particles" are growing based of the sum of the frequencies, the louder an audio frame, the bigger the particles.
The user can choose between non-copyrighted preset songs or can even upload their own song. If a song has ended, the next song will be played.
The user can also choose the displaying mode and if they want to skip the song.
