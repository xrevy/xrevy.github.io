Forgotten Darkness

SS 2021

Forgotten Darkness was a Bachelor's Project, that was created in a group of 5. The main theme was "Nachtmahr" (Nightmar) and this project was supposed to be a story-driven horror game, in which you play a person with amnesia, who wakes up in a huge mysterious villa.

Every interface sprite is selfmade, I used Procreate for that, the music and a few soundeffects are also made by our group, like objects like the whole house, the ravens to name a few.
The 3D models were created in Blender, we worked in the Unreal Engine. Character models were created by either the unreal technology Metahuman or DAZ (for the children models).
My main task in this project was programming basic game functions and programming general game elements. Apart from that I designed the protagonist via "Metahumans", imported it into the game an added its functions, but I also created all the 2D game sprites, newspaper and created the sound of the radio announcements.
The biggest Problem we ran into was the performance, we had to implement a better level/sequence streaming system for our game in order to make some certain sequences better playable.

Webcam Hero

SS 2022

This project was initialized as a product that uses motion tracking in some way. I wanted to create a game that utilizes that. In the beginning I started developing in Unreal Engine 5, but had soon to dicover that I ran into problems with kernel access, so a quick decision had to be made.

After some research I came across a solution for another game engine that I've already used before: Godot. This project contains two projects, one godot game project and one python motion detection that sends its output via a local UDP server to the game in process.
The program searches for faces and compares the current face position on the y-axis with the previous y-position. If the current position is higher than the previous one, the application sends the signal for jumping, otherwise for crouching. If the threshold was not reached, the signal for "do nothing" will be sent.
The game itself is a jump and run game that uses kamera input as a controller, if the player jumps, the game character jumps, same for crouching.
The player character is an astronaut that has to evade octopus aliens. The game is set on the moon.
The game assets were also made by me, I used aseprite to draw the pixelart. Just the music and the sound effects are downloaded CC0 files.

Magical Quarters

WS 2020/21

The first game project in general and the first time I got to use Unreal Engine. This project was supposed to teach us how to use Unreal Engine.

We worked in a group of seven. First, in order to begin this project, we created our game concept, followed by a detailed requirements analysis. Once that was done, we started planning our game application. We used different of UML diagrams to plan the different aspects of the game (a few diagrams are seen in the carousel below).
So our game should be a 3D rogue-like game, with different rooms, that get locked once the player enters it to clear it from enemies. In the end the player hast to defeat a boss in order to win the game.
Game assets like player character and enemy 3D models were created by us just like game icons, loading screens and background pictures.
This project teached us a lot about project preparation and planning, game development and team composition. While we had to manage who has to do what, we suddenly faced the problem of missing three teammates further in the development phase so we had to adjust our plannings.
